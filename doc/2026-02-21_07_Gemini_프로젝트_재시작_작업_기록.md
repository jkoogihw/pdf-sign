# 로컬 프로젝트 재시작 작업 기록 (2026-02-21)

로컬 개발 환경에서 프로젝트 서버를 확인하고 재시작한 과정을 기록합니다.

## 1. 프로젝트 상태 점검 방법
서버가 실행 중인지 확인하기 위해 다음 명령어를 사용할 수 있습니다.

### 포트 점검 (PowerShell)
```powershell
# 8080 포트를 사용 중인 프로세스 확인
netstat -ano | findstr :8080
```
- 결과가 나온다면 마지막 숫자가 해당 프로세스의 **PID**입니다.

### 프로세스 상세 확인
```powershell
# 특정 PID에 대한 프로세스 정보 확인 (예: PID 1234)
Get-Process -Id 1234 | Select-Object Id, ProcessName, CommandLine
```

## 2. 재시작 작업 기록
PowerShell 스크립트 실행 시 권한 이슈(`UnauthorizedAccess`)가 발생하여, 실행 정책을 우회하여 서버를 시작하였습니다.

### 실행 명령어
```powershell
# 권한 우회 명령어로 서버 실행
powershell -ExecutionPolicy Bypass -File server.ps1
```

### 서버 구성 정보
- **서버 주소**: [http://localhost:8080](http://localhost:8080)
- **정적 파일 경로**: `D:\dev\repository\git.jkhw\pdf-sign` (프로젝트 루트)

## 3. 실행 결과 확인
- **서버 로그**: "Server running on http://localhost:8080" 메시지 확인.
- **접속 테스트**: `http://localhost:8080/index.html` 호출 시 정상 페이지 로드 확인.

이후 프로젝트 상태를 확인하거나 재시작이 필요할 때 다음 가이드를 참고하세요.

> **작업 지침:**
> 1. 현재 로컬 서버(Port 8080)가 실행 중인지 `netstat -ano | findstr :8080` 명령어로 확인하세요.
> 2. 만약 실행 중이라면 `Get-Process`를 통해 `server.ps1`이 맞는지 검증하세요.
