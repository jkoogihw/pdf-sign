# 2026-02-21_06_Gemini_드래그앤드롭_업로드_및_자동_검증_환경_구축.md

## 1. 개요
- 사용자 편의성을 높이기 위한 드래그 앤 드롭 UI를 고도화하고, 시스템의 신뢰성을 보장하기 위해 실제 보험 양식을 활용한 자동 검증 환경을 구축함.

## 2. 주요 작업 내용

### 2.1. 드래그 앤 드롭(Drag & Drop) 고도화
- **2열 레이아웃 적용**: 업로드 섹션 우측에 전용 드롭 영역을 배치하여 직관적인 조작 가능.
- **실시간 피드백**: 
    - PDF 파일 드롭 시 **파일명**을 즉시 표시.
    - 서명 이미지 드롭 시 **이미지 미리보기(Thubmnail)**를 즉시 렌더링.
- **자동 교체 로직**: 파일이 이미 있는 상태에서 다시 드래그하면 기존 파일이 새 파일로 즉시 교체되도록 구현.

### 2.2. 샘플 데이터 자산화 (Asset Management)
- 실제 업무에서 사용되는 3종의 보험 양식과 서명 이미지를 프로젝트 내 정식 샘플로 등록함.
- **경로**: `samples/`
    - `1_long_term.pdf`: 장기보험 양식
    - `2_general.pdf`: 일반보험 양식
    - `3_auto.pdf`: 자동차보험 양식
    - `sign.png`: 검증용 서명 이미지

### 2.3. 자동 검증 테스트 페이지 (`test.html`) 구축
- **목적**: 코드 수정 시 핵심 탐색 로직이 정상 작동하는지 수동 검증 없이 즉시 확인.
- **기능**:
    - `samples/`의 PDF를 자동으로 분석하여 양식 감지 결과(장기/일반/자동차) 리포팅.
    - 텍스트 좌표(X, Y) 및 너비 계산 결과 표시.
    - **결과 PDF 다운로드** 버튼을 통해 실제 서명 배치 상태를 시각적으로 즉시 확인 가능.

### 2.4. 가독성 개선 및 안내 문구 최적화
- 화면 상단의 하드코딩된 이름('조정국')을 제거.
- 시스템이 제공하는 실제 기능(양식별 자동 감지 및 배치)을 명확하게 설명하도록 텍스트 정규화.

## 3. 향후 유지보수 가이드
- 코드를 수정한 뒤에는 반드시 `http://localhost:8080/test.html`에 접속하여 모든 케이스가 `[PASS]` 인지 확인하십시오.
- 새로운 보험 양식이 추가될 경우 `samples/` 폴더에 파일을 넣고 `test.html`의 `testCases` 배열에 추가하여 검증을 자동화할 수 있습니다.
