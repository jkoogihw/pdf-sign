# 2026-02-21_02_Gemini_로컬_서비스_캐시_이슈_해결_및_재실행.md

## 1. 문제 현상
- 로컬 서비스를 중단했음에도 브라우저에서 `http://localhost:8080` 접속 시 이전 화면이 계속해서 나타남.
- `netstat` 확인 결과 8080 포트를 점유 중인 프로세스가 없으나, 브라우저는 마치 서비스가 구동 중인 것처럼 동작함.

## 2. 원인 분석
- **브라우저 캐시 및 Service Worker**: 이전에 구동되었던 서비스가 PWA(Progressive Web App) 속성을 가지고 있거나, Service Worker가 등록되어 있을 경우 오프라인에서도 화면을 표시하기 위해 데이터를 캐싱함.
- **오프라인 캐싱**: 네트워크 연결이 끊겨도 로컬에 저장된 리소스를 사용하여 화면을 렌더링하기 때문에 발생하는 현상임.

## 3. 조치 내용 (브라우저 데이터 초기화)
브라우저에 남은 캐시와 등록된 Service Worker를 완전히 제거하기 위해 다음 단축키 및 절차를 수행함.

### 3.1. 강력 새로고침 (Hard Refresh)
- **Windows**: `Ctrl` + `F5`
- 캐시를 무시하고 서버에서 새로운 리소스를 요청함.

### 3.2. 개발자 도구를 통한 사이트 데이터 삭제
1. `F12`를 눌러 개발자 도구 실행.
2. **Application (애플리케이션)** 탭 선택.
3. 좌측 메뉴에서 **Storage (스토리지)** 클릭.
4. **Clear site data (사이트 데이터 삭제)** 버튼 클릭. (Unregister service workers 포함)

## 4. 로컬 서비스 재실행
서비스를 다시 구동하여 정상 동작 여부를 확인함.

### 실행 및 점검 명령어
```powershell
# 8080 포트 사용 여부 확인
netstat -ano | findstr :8080

# 로컬 서버 실행
powershell -ExecutionPolicy Bypass -File .\server.ps1
```

### 결과 확인
- 터미널 로그: `Server running on http://localhost:8080` 확인.
- 브라우저에서 `http://localhost:8080` 접속 시 최신 버전의 PDF 서명 서비스가 로드됨.
